Seção 1

Introdução

O que é Programação Dinâmica?

Técnica algorítmica que decompõe problemas complexos em subproblemas menores, resolvendo cada um uma única vez e armazenando os resultados para consulta futura.

Objetivo do Trabalho

Investigar a aplicação da DP ao problema da Mochila 0/1, consolidando a compreensão teórica e demonstrando a aplicação prática em JavaScript.

Por que a Mochila 0/1?

Problema canônico da otimização combinatória (NP-difícil) que exemplifica perfeitamente as duas propriedades fundamentais da DP: subestrutura ótima e sobreposição de subproblemas.

Seção 2

Fundamentação Teórica

Duas Propriedades Essenciais

Subestrutura Ótima

A solução ótima do problema contém as soluções ótimas de seus subproblemas.

Sobreposição de Subproblemas

O mesmo par (i, w) é visitado repetidamente — evitamos recalcular com tabela.

Memoização vs Tabelamento

Memoização (Top-Down)

Recursão com cache. Resolve apenas subproblemas necessários.

Tabelamento (Bottom-Up) ✓

Iterativo, preenche a tabela dos casos base até o resultado final. Adotado neste trabalho.

Richard Bellman (1950) — Princípio da Otimalidade

O Problema

Mochila 0/1 — Formulação

Dados n itens com peso wᵢ e valor vᵢ, e capacidade W:

Maximizar ∑(vᵢ · xᵢ) , sujeito a ∑(wᵢ · xᵢ) ≤ W , xᵢ ∈ {0, 1}

Decisão binária: levar (1) ou não levar (0)

2ⁿ

Força Bruta

Inviável para n > 40

n · W

Programação Dinâmica

Solução exata pseudo-polinomial

≈

Algoritmo Guloso

Não garante solução ótima

Recorrência

Definição da Recorrência

Seja dp[i][w] o valor máximo considerando os primeiros i itens com capacidade w:

dp[i][w] = max( dp[i−1][w] , dp[i−1][w−wᵢ] + vᵢ )
se wᵢ ≤ w | senão: dp[i][w] = dp[i−1][w]

→ Não incluir

Herda o valor da linha anterior com mesma capacidade

→ Incluir

Soma o valor do item à melhor solução com capacidade reduzida

Casos base: dp[0][w] = 0 dp[i][0] = 0

Backtracking

dp[4][8] = 10 → difere de dp[3][8] = 9 → Item 4 incluso. w = 3. dp[3][3] = dp[2][3] → Item 3 não. dp[2][3] ≠ dp[1][3] → Item 2 incluso. w = 0.

Solução Ótima

Itens: {Item 2, Item 4}
Peso: 3 + 5 = 8 ton (100%)
Valor: R$ 10.000

Implementação

Duas Versões do Algoritmo

Versão Matricial O(n·W) espaço

function mochila01(pesos, valores, cap) {
  const n = pesos.length;
  const dp = Array.from(
    { length: n+1 },
    () => Array(cap+1).fill(0));
  const keep = Array.from(
    { length: n+1 },
    () => Array(cap+1).fill(false));
  for (let i = 1; i <= n; i++) {
    for (let w = 0; w <= cap; w++) {
      if (pesos[i-1] <= w) {
        const sem = dp[i-1][w];
        const com = dp[i-1][w-pesos[i-1]] + valores[i-1];
        if (com > sem) {
          dp[i][w] = com;
          keep[i][w] = true;
        } else dp[i][w] = sem;
      } else dp[i][w] = dp[i-1][w];
    }
  }
  // backtracking...
}

Versão Otimizada O(W) espaço

function mochila01Otimizada(pesos,valores,cap){
  const n = pesos.length;
  const dp = Array(cap+1).fill(0);
  const keep = Array.from(
    { length: n+1 },
    () => Array(cap+1).fill(false));
  for (let i = 1; i <= n; i++) {
    // Loop DECRESCENTE
    for (let w = cap; w >= pesos[i-1]; w--) {
      const com = dp[w-pesos[i-1]] + valores[i-1];
      if (com > dp[w]) {
        dp[w] = com;
        keep[i][w] = true;
      }
    }
  }
  // backtracking (igual)...
}

⚠ Loop decrescente evita reutilizar o mesmo item múltiplas vezes

Análise

Complexidade Computacional

Complexidade de Tempo

O(n · W)
  • ✓Preenche (n+1) × (W+1) células
  • ✓Cada célula: O(1) operações
  • ✓Classificação: pseudo-polinomial

Complexidade de Espaço

MatricialO(n · W)

dp[n+1][W+1] + keep[n+1][W+1]

OtimizadaO(W)

dp[W+1] (vetor único) + keep

⚠ Pseudo-polinomial: a complexidade depende do valor de W, não do número de bits.
